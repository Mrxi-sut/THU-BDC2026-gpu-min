# 1、改动点
加入组合收益型 loss
现在训练主要是 listwise/pairwise 排序损失。可以增加一个 soft Top-K 组合收益损失，例如让模型输出分数 score，通过 softmax 或 differentiable top-k 近似成权重 w，然后最大化：
portfolio_return = sum(w_i * future_return_i)
# 2、改动点
把金融特征按语义分组，再用市场状态动态门控论文不是把所有技术指标直接丢进 Transformer，而是把 95 个特征分成价格、收益、波动率、成交量、K线形态、滞后项、突破、日历等 11 类，再由 Market Regime Encoder 判断当前市场状态，动态调整各组特征权重。这个优点很适合金融数据，因为不同市场环境下有效信号不同：趋势行情更依赖动量/突破，震荡或高波动阶段更依赖波动率、回撤、量价确认。
但深度模型侧仍是统一线性投影所有输入特征，见 [code/src/model.py (line 64)](/Users/zx/Desktop/THU-BDC2026-gpu-min/code/src/model.py:64)。论文的“语义分组 + 门控”可以直接补强你当前 Transformer 分支。
# 3、改进点
优先做“表格 selector 的学习型 regime gate”，风险最低。你现在的最终主信号是 selector_score，表格分支权重很高，tabular_blend_weight=0.95，见 [code/src/config.py (line 49)](/Users/zx/Desktop/THU-BDC2026-gpu-min/code/src/config.py:49)。所以最划算的改进不是先大改 Transformer，而是把 selector_consensus、selector_hot_rotation、selector_defensive 的规则切换改成轻量学习门控。
当前 add_strategy_selector_score 是规则分型后手工切换。可以新增一个小模型或逻辑回归/HistGradientBoosting gate，输入：market_ret_5、market_ret_20、up_ratio_5、up_ratio_20、market_drawdown_20、ret dispersion、amount_ratio、top2_vs_top10_gap、selector_hgb_overlap
输出三个权重：w_consensus、w_hot_rotation、w_defensive
最终：selector_score = w1 * selector_consensus+ w2 * selector_hot_rotation+ w3 * selector_defensive
这样等价于论文的 Adaptive Gate Network，但落在你最有效的表格主干上。
